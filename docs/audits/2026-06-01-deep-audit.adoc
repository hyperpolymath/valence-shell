== Newly surfaced drift (not in issue #44 ‚Äî found during execution)

These were discovered while executing the #44 plan and are recorded
here rather than the PR description (per protocol):

. **`README.adoc:3` license badge URL is malformed** ‚Äî was
  `image:https://[License: PMPL-1.0,link="..."]`, which is not a valid
  AsciiDoc image macro. Fixed to a proper `shields.io` URL with the
  correct `MPL-2.0` licence (the PMPL-1.0 reference was also stale
  ‚Äî the project migrated to MPL-2.0 in commit `990b147`).

. **`README.adoc:540` citation BibTeX claimed version `0.14.0`,
  year `2025`** ‚Äî out of sync with the rest of the document (which
  is consistently `0.9.0` / 2026). Fixed.

. **`README.adoc:490` FAQ "production-ready" answer referenced
  `v0.14.0`, `82% complete`, `177 tests`** ‚Äî three independent
  staleness signals in the same paragraph. Reconciled to v0.9.0,
  ~78%, 736 tests.

. **`0-AI-MANIFEST.a2ml` is a template stub** referring to
  `rsr-template-repo` and `.machines_readable/6scm/` (a path that
  does not exist; canonical is `.machine_readable/`). Reconciled in
  this audit cycle to point at the actual contractile family layout.

. **`MAINTAINERS.md` content is materially different from
  `MAINTAINERS.adoc`** ‚Äî the `.md` file lists a "[To be added]"
  contact placeholder; the `.adoc` file lists Jonathan D.A. Jewell
  as Lead Maintainer with GitHub handle. Choosing the `.adoc` (more
  current + estate convention) ‚Äî see the `Deleted` section above.

. **`CHANGELOG.md` and `CHANGELOG.adoc` had structurally different
  histories** ‚Äî the `.md` file ended at `0.14.0` (2026-01-29);
  the `.adoc` file includes the same history but at full detail
  with proper `// SPDX-License-Identifier: CC-BY-SA-4.0` header. The
  `.md` is also missing several entries that the `.adoc` has.
  Keeping `.adoc` (estate convention + more complete).

. **README `Status:` footer line said `Tests: 602 passing`** but
  the `IMPORTANT` admonition near the top already showed the same
  `602` figure, while `CLAUDE.md` has been at `736` since 2026-04-20.
  Drift confirms #44's diagnosis.

. **Wiki state (per `gh api repos/hyperpolymath/valence-shell --jq
  .has_wiki`):** wiki is **enabled but effectively empty** ‚Äî single
  stub page `Home.md` containing only "Welcome to the valence-shell
  wiki!". Surfaced here as a follow-up rather than written from
  this audit (wikis are user-facing artefacts and need owner
  direction on tone/scope). Potential follow-up: decide whether to
  (a) populate with curated extracts of `README.adoc` + `FAQ.adoc`,
  (b) disable the wiki and redirect readers to `docs/`, or (c)
  leave as-is with a `Home.md` pointing at `docs/` and
  `EXPLAINME.adoc`.

. **`impl/rust-cli/$FILE`** ‚Äî a literal 5-byte file named `$FILE`
  exists at the top of `impl/rust-cli/`. Almost certainly a shell
  artefact from a CI cycle (`$FILE` unset, command wrote to it

== Methodology used

This audit followed the verisimdb#81 shape:

. Enumerate canonical-file presence + currency (the per-file table
  at the top of issue #44).
. Locate placeholder strings (`{{...}}`) in `.machine_readable/`.
. Cross-check numeric claims (theorem count, test count, source-file
  count, fuzz-target count, proof-hole count) against actual reality
  via `find`/`wc -l`/`grep` and the `PROOF_HOLES_AUDIT.md`.
. Diff dual-source pairs (`MAINTAINERS.adoc` vs `.md`, `CHANGELOG.adoc`
  vs `.md`) and consolidate on the estate-convention winner.
. Walk the open-issue tail (#41‚Äì#46, #49, #51) and capture each as
  audit-table input.
. Inspect sibling-repo `.machine_readable/bot_directives/methodology.a2ml`
  (echidna, proven, ephapax) to ground the divergence-statement in
  `methodology.a2ml`.
