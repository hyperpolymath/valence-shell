## Maintainer Responsibilities

### Core Maintainers (Perimeter 1)

**Authority**:
- Approve/reject Perimeter 1 changes (formal proofs)
- Merge Perimeter 2 contributions (implementations)
- Set technical direction
- Manage releases

**Duties**:
- Review pull/merge requests
- Maintain formal proofs quality
- Ensure proof-implementation correspondence
- Security response
- Community moderation (Code of Conduct)

### Trusted Contributors (Perimeter 2 Access)

**Authority**:
- Submit directly to Perimeter 2 (with review)
- Review Perimeter 3 contributions
- Merge Perimeter 3 after review

**Duties**:
- Implement features matching formal specs
- Write tests and documentation
- Review community contributions
- Help newcomers

### Community Contributors (Perimeter 3 Access)

**Authority**:
- Submit to Perimeter 3 freely
- Create examples and tutorials
- Improve documentation

**Responsibilities**:
- Follow Code of Conduct
- Test contributions
- Document work
- Be helpful to others

## Becoming a Maintainer

### Path to Trusted Contributor (Perimeter 2)

1. **Track Record**: 5+ quality contributions to Perimeter 3
2. **Understanding**: Demonstrate grasp of formal specifications
3. **Consistency**: Follow coding standards and review process
4. **Community**: Help others, participate constructively
5. **Nomination**: Current maintainer nominates you
6. **Vote**: Existing maintainers vote (majority required)

### Path to Core Maintainer (Perimeter 1)

1. **Expertise**: Proven formal verification proficiency
2. **Trust**: 1+ year as Trusted Contributor
3. **Proofs**: Contributed significant proofs or verified implementation
4. **Security**: Understanding of security implications
5. **Nomination**: Core maintainer nominates you
6. **Consensus**: All core maintainers must approve
